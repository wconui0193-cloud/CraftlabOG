## Session 2 — GSAP Animation Fixes (Scroll Restoration + ScrollTrigger.refresh Removal)

---

### A. Session Summary

After Session 1 was saved, the master reported that animations were breaking on F5 refresh — specifically the keychain orbit carousel was becoming static, and other GSAP scrub animations were not initializing correctly. Root cause was traced to two issues: `window.scrollTo(0, 0)` in the deferred script.js was executing while GSAP was setting up pins, causing layout conflicts; and a `ScrollTrigger.refresh()` call in the hash nav fix was recalculating positions incorrectly after the journey section pin spacers changed. Both were removed and the scroll restoration was moved to a synchronous inline `<head>` script.

---

### B. Changes This Session

**index.html**
- Added inline `<script>` tag in `<head>`: `<script>if ('scrollRestoration' in history) history.scrollRestoration = 'manual';</script>` — runs synchronously before any deferred scripts, prevents browser from restoring scroll position before GSAP initializes
- script.js bumped from v110 to v112

**script.js (v112)**
- Removed `if ('scrollRestoration' in history) history.scrollRestoration = 'manual';` from the top of the file — moved to inline `<head>` script in index.html
- Removed `window.scrollTo(0, 0)` from the top of the file — was executing during GSAP init, corrupting pin spacers and scrub state
- Removed `ScrollTrigger.refresh()` call from the HASH NAV FIX IIFE — was recalculating positions incorrectly after journey section spacers changed, causing sticky card stacking section to show intro slide and card 1 simultaneously
- Hash nav fix (bottom of script) now does a simple smooth scroll to target after 350ms delay — no refresh needed because scroll always starts at 0

**Bugs fixed**
- Keychain orbit animation was becoming static on F5 — caused by `window.scrollTo(0, 0)` in deferred script running while GSAP was setting up ScrollTrigger for the orbit. Removed.
- Sticky card stacking section showed intro slide and card 1 simultaneously after hash nav — caused by `ScrollTrigger.refresh()` in hash nav fix recalculating spacers incorrectly. Removed.
- All animations now initialize correctly from scroll=0 on every page load/refresh. Master confirmed: "perfect."

### L. Technical Notes

**CRITICAL — Animation Initialization Rules:**
- `history.scrollRestoration = 'manual'` MUST be in an inline `<script>` in `<head>` — NOT in a deferred script. Deferred runs too late; browser has already attempted scroll restoration by then.
- NEVER use `window.scrollTo(0, 0)` or `document.documentElement.scrollTop = 0` while GSAP ScrollTrigger is active — corrupts pin spacers and scrub state. Full page reload with hash is the correct pattern.
- NEVER call `ScrollTrigger.refresh()` in any event handler (visibilitychange, hash nav, or otherwise) — recalculates positions incorrectly when journey pin spacers are in a changed state, breaks sticky card stacking and scrub animations.
- The hash nav fix works WITHOUT refresh because scroll always starts at 0 (scroll restoration is disabled). GSAP initializes at scroll=0, so no recalculation is needed.
