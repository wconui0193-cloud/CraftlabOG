### L. Technical Notes

**CSS Custom Properties (`:root`):**
```css
--orange:   #FF6A00
--gold:     #D6A14A
--white:    #F6F7F8
--gray:     #707782
--graphite: #1A1A1A
--dark:     #111111
--black:    #0B0B0E
--font-h:   'Barlow Condensed', sans-serif
--font-b:   'Montserrat', sans-serif
--ease-out:    cubic-bezier(0.23, 1, 0.32, 1)
--ease-in-out: cubic-bezier(0.77, 0, 0.175, 1)
--ease-drawer: cubic-bezier(0.32, 0.72, 0, 1)
```

**Responsive breakpoints:**
- `≤960px` — hero splits stack vertically
- `≤900px` — container padding shrinks, problem grid goes 1-col, outcomes 1-col, `.prob-carousel` loses sticky
- `≤600px` — section padding reduces, flow strip scrolls horizontally, CRM preview goes 1-col
- `≤560px` — nav strip shrinks to 56px, body padding-left: 56px
- `≤480px` — hero adjustments
- `≤400px` — nav strip 52px, various small phone fixes

**Scroll architecture:**
- Statement carousel: 600vh wrapper + `position: sticky; height: 100vh` — scroll progress 0→1 maps to slide 0→5
- About parallax: `getBoundingClientRect().top / window.innerHeight` × 80px = translateY shift
- Section active tracking: `window.scrollY >= sec.offsetTop - 200` threshold

**Animation conventions (from Emil Kowalski skill influence):**
- Stagger delays: 30–80ms between items
- Hover transitions: guarded by `@media (hover: hover) and (pointer: fine)`
- Reduced motion: `prefers-reduced-motion` collapses all to 0.01ms
- `will-change: transform` on `.stmt-track` only

**Font override pattern for nav:**
Nav headings are explicitly set to `'Montserrat', sans-serif; text-transform: none` because the global `h1, h2, h3` rule applies Barlow Condensed uppercase. Any new heading-level elements inside `.nav` must have this override applied.

**Prob-carousel card stacking pattern:**
Cards overlap via CSS grid trick — all `.prob-card` share `grid-area: stack` inside `.prob-track { grid-template-areas: 'stack' }`. Active card gets `z-index: 1`. Non-active cards are `opacity: 0; pointer-events: none` but still occupy space (sets container height to tallest card).

**Progress bar pattern (prob-carousel):**
Reset by setting `transition: none` + `width: 0%` + forced reflow (`void bar.offsetWidth`) then re-setting `transition: width ${DURATION}ms linear` + `width: 100%`. Works reliably across cards.
